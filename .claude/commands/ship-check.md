# /ship-check — Pre-push gate

This mirrors `.github/workflows/azure-static-web-apps-witty-grass-0a1a9d403.yml`.
Run it locally before every `git push`. Stop on the first failure and fix the
root cause — never bypass.

Execute each command in order. After any failure, do **not** continue; fix and
restart from step 1.

```bash
npm run check-logo-provenance
npm run check-logo-audit-report
npm run check-color-literals
npm run lint
npx tsc --noEmit
npm run check-governance-data
npm run check-updates-data
npm run test:eu-ai-act
npm run build
npm run check-generated-artifacts
npm run check-open-data
npm run check-seo-readiness
npm run check-tool-card-data
npm run check-deploy-readiness
```

`check-deploy-readiness` is the one local-only step: it verifies live deploy
plumbing (Azure SWA connection, secrets, a green deploy on main) that the CI
job itself cannot meaningfully self-inspect. Every other step above runs in CI
in the same order.

If `npm run build` regenerated SEO artifacts under `app/` or `public/`, those
diffs must be committed before pushing — otherwise `check-generated-artifacts`
will fail on CI even after it passed locally.

If `npx tsc --noEmit` reports errors in files outside this PR's scope, surface
to the user — do not silently fix unrelated drift in the same PR.

Report at the end: which steps ran, which passed, and the diff summary of any
files regenerated by `npm run build`.
